js高级知识点集合

JS高级

  1. 原型的概念

    • 什么是原型？函数的prototype属性所引用的对象
    • 原型的本质是对象 当创建函数时 原型就同时产生 是早于对象的创建
    • ★ 四个特性
      • 动态性
        • 给原型扩展成员会直接反应到已创建的对象身上
        • 替换原型对象，不会反映到已创建出来的对象。但是会直接影响之后创建出来的对象。
      • 唯一性
        • 由同一函数创建出来的所有对象，共享同一个原型对象。
      • 不可变型
        • 对象是无法改变原型对象上的任何成员
      • 继承（共享）性
        • 所有对象都继承自它的原型对象
    • 如何区分 __prototype__ prototype constructor
      • 对象才有__prototype__ 属性
      • prototype 的函数原型 Function
      • constructor

  2. 继承的概念

    • 指一个对象有权去访问两个对象上的成员
    • 好处： 实现代码复用

      • ★ 原型式：扩展原型 和 置换原型
      • ★ 拷贝继承（混入式）：利用拷贝一个对象上的所有成员添加到自身上 来实现继承

  6. 闭包 本质是函数 或者说是表达式

    • 概念：一个嵌套函数 函数内的变量能访问另个函数表达式

      • 指那些能够访问独立（自由）变量的函数（表达式）（变量在本地使用，但定义在一个封闭的作用域中）换句话说，这些函数可以‘记忆’它被创建时候的环境。

    • 应用

      • 缓存数据
      • 私有属性
      • 沙箱模式
      • 提升递归性能
      • 柯里化应用

词法作用域与闭包

  1. 熟练使用词法作用域规则

  2. 闭包的理解

    • 一般闭包的结构 为 函数嵌套函数
    • 内部函数对外层函数的变量有引用，导致被引用的变量不会被垃圾回收机制销毁掉，
      从而形成缓存数据的效果。
    • 每一次调用外层函数，都会产生新的闭包。
    • 不同的闭包内部缓存的数据是独立的。
    • 想要修改以及获取闭包中的数据，必须在闭包中提供相关方法才可以。

  3. 函数四种调用模式

    • 作用：确定函数内部的this指向问题
    • 普通函数执行： this 指向 window （非严格模式）
    • 方法调用模式： this 指向 方法的调用者，即谁调用就是谁。
    • 构造函数模式： this 指向 该构造函数所创建出来的实例。

    • 上下文模式

      • fn.call(thisObj, arg1, arg2, …,argN) 或 fn.apply(thisObj, [fn的实参列表]);
      • 当函数掉用call或apply方法时，函数就开始执行了。
      • 通过call和apply可实现动态改变fn内部this的指向，而两个方法中的thisObj均为 新的this指向。
      • call和apply的区别在于：给函数fn执行时传入实参的方式不同，call方法是以arg1到argN的可选形参列表方式；而apply是以数组中的每个元素作为实参传给fn的。
      • 在选择使用call还是apply时，就看哪个方法用时方便。

    • 注意：在每个函数内部的this 不会相同。所以在使用this时，时刻要分清this的指向。

  4. 闭包的应用：在闭包的所有应用领域，都是基于闭包可以缓存数据的特性。

    • 沙箱模式：防止污染其他作用域变量，同时保证本身代码的正确性。将部分接口暴露在全局，
      供其他用户使用。
    • 其他: 基于闭包的缓存数据特性的应用，包括计数器，解决递归性能等。

继承的概念

  1. js并不是一门面向对象的语言，而是基于对象的。但是由于面向对象的思想有诸多好处：可读性、可维护性、可扩展性。
    因此在js中，尝试着去模拟面向对象编程代码结构。也正是由于原型的存在，得以让继承可以在js中大放异彩。

  2. 而在js中，所有能使一个对象有权去访问另一个对象的所有成员，都被称为继承。

  3. 也是因此，在js中继承模式种类繁多，但是常用的几个要认得：比如原型式、混入（拷贝）式、对象冒充、借用构造函数

DOM事件流

  1. 概念：DOM结构是一个树型结构，当一个HTML元素产生一个事件时，该事件会在元素结点与根结点之间的路径传播，路径所经过的节点都会收到该事件，这个传播过程可称为DOM事件流。

  2. DOM同时支持两种事件模型：捕获型事件和冒泡型事件，但是，捕获型事件先发生。两种事件流会触发DOM中的所有对象，从document对象开始，也在document对象结束。

绘制作用域链规则

1.将整个script标签的全局作用域，定义为0级作用域链。将全局作用域上的数据（变量、函数、对象等）绘制在该链上。

2.由于在词法作用域中，只有函数可以分割作用域。因此，只要遇到函数，就引申出新的作用域链。级别为当前链的级别+1.

3.重复第二个步骤，直到没有遇到函数。

变量搜索原则

当访问一个变量时：

1、首先在当前变量所处作用域链上查找，如果找到就直接使用，停止查找。

2、如果没有找，就向上一级链（T-1）上查找，如果找到直接使用，通知查找。

3、没有找到继续向上，直到0级，找到后，使用

4、再次没有找到就报错。

注意：

1、如果访问的变量不存在，会搜索整个作用域链（不仅能性能低，而且抛出异常）

  • 在实际开发时，不推崇所有的数据写在全局上。尽量使用局部变量。
  • 如果在开发中，所有js代码都写在全局上，会造成全局变量的污染。

2、同级别的链上的变量互不干扰。

变量提升

在函数执行 的时候，会创建一个执行的环境，包括activeObeject（活动对象）和作用域链。

活动对象存储传进来的值（变量、对象、函数等），变成自己的属性。但是如果多次传入变量并赋值，会更改自己的属性。是一个单向链，多次向上寻找。

1、活动对象存储的是所有在函数内部定义的变量，以及函数形参。

会将变量的名字以及形参名字作为该对象的属性来存储。
访问时，向上一级链查找，遇到重复的形参或者变量名字的时候，会替换值。

2、解析时，浏览器跟踪不到。
