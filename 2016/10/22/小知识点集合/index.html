1. 出现false的情况

  • 空字符串、false、undefined、null、0

2. 出现undefined 的情况

  • 变量声明未赋值
  • 语句没有返回值
  • 函数没有传参
  • 访问数组不存在的项
  • 对象没有该属性
  • 函数没有返回值 （return）

4. 数据类型之间的比较

5. 变量的类型转换

string类型

  • 强制转换
    • 变量.toString()
    • String()
      • undefined 和 null 不支持 toString() 方法 报错
      • undefined 和 null 支持 toString() 方法
  • 隐式转换
    • 通过任意类型 + 任意字符串操作转换为 字符串类型
    • +的时候为了不改变数据原有的内容，可以使用''字符串
    • 空字符串不能有任何内容

number类型

  • 强制转换
    • Number() 只能转换全部是数值型的字符串 Number('123') -> 123
    • parseInt() 可以转换前半部分为数字的字符串，转换结果为整数 parseInt('123.5abc') -> 123
    • parseFloat() 可以包含小数 parseFloat('123.5abc') -> 123.5
  • 隐式转换
    • - * / % 可以进行转换
      • 操作放在数据前使用

bool类型

  • 强制转换
    • Boolean()
      • 转换值为 false 的情况：
        • 0
        • undefined
        • null
        • ‘’
        • NaN
  • 隐式转换
    • !!

undefined和 null 只有一个值 不需要转换

6.短路操作

  • 短路操作的法则：作为&&和||操作符的操作数表达式，这些表达式在进行求值时，只要最终的结果已经可以确定是真或假，求值过程便告终止。通俗的说，隐式转换完毕后，哪个操作数决定表达式的结果，就返回谁

  • 逻辑运算符

    • 且 &
    • 或 ||
    • 非 !

7.运算符的优先级

优先级从高到低

  • () 优先级最高
  • 一元运算符 ++ – ！
  • 算数运算符 * / % + -
  • 关系运算符 > >= < <=
  • 相等运算符 == !== === !===
  • 逻辑运算符 && ||

8.条件判断语句

通常情况下我们使用if else进行范围比较，使用switch case进行单值比较。

  • if…else…

if 语句是 ECMAScript 中最常用的语句之一，事实上在许多计算机语言中都是如此。

  • if 语句的语法：
  1. 其中 condition 可以是任何表达式，计算的结果甚至不必是真正的 boolean 值，ECMAScript 会把它转换成 boolean 值。
  2. 如果条件计算结果为 true，则执行 statement1；如果条件计算结果为 false，则执行 statement2。
  3. 每个语句都可以是单行代码，也可以是代码块。

还可以多个串联

  • switch case

9.判断数据类型方法

  • typeof
    • 函数 -> function
    • 布尔 -> boolean
    • 数组 -> object
    • 空值 null -> object
    • 字符串 -> string
    • 数字 -> number
    • 未定义 undefiend -> undefiend
  • Object.prototype.toString.call
    • console.log( Object.prototype.toString.call(Array()) ) -> [object Array]
    • console.log( Object.prototype.toString.call(null) ); -> [object null]
