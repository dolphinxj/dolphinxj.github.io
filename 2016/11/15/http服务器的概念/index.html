http服务器的概念

http

  • http是什么？
    • 就是一种协议
    • http本身就是一个对数据格式约定的协议
      • 浏览器和服务器之间的约定
      • 前后端接口文档
    • http遵循一问一答的机制
  • http的五大特点
    • 支持客户端与服务端架构
    • 简单
    • 灵活
    • 无连接 服务器不为客户端保留链接，响应后立即关闭链接（短链接和长链接）
    • 无状态 服务器不去记录某个在用的状态（减轻服务器的内存压力）cookie的技术 基于cookie的session

socket

他机连接本机程序 使用ip和端口
本机查找程序 使用pid更方便

HTTP核心模块相关对象

  • 服务器对象（sever）
    • 创建服务器对象: http.createServer(fucntion(req,res){});
    • 开启服务，监听端口 server.listen(port,host,log,callback);
    • 事件: server.on(‘request’,function(req,res){..})
  • 客户端对象
    • options = {protocal:’http’,host:’www.itcast.cn’,port:8080,path:’/‘};
    • 返回客户端请求对象 http.request(options,function(res){….});
    • 请求对象.end(发送内容);

  • 请求对象

    • 获取用户请求资源路径: req.url
    • 获取用户请求方式: req.method
    • 获取请求头数据 req.headers 返回一个对象，通过key来获取值
    • 获取数据事件： req.on(‘data’,function(data){..操作请求数据..}); req.on(‘end’,function(){接受数据完毕})

  • 响应对象

    • 先设置头信息、再写头信息、再写信息、再结束写信息
    • res.setHeader(key,value);
    • res.writeHead(状态码,{‘Content-type’:’text/html;charset=utf-8’});
    • res.write(‘字符串和二进制数据’,编码:默认utf-8);
    • res.end(‘字符串和二进制数据’,编码:默认utf-8);

思想及理论

  • SYN(synchronous) 建立联机
  • ACK(acknowledment) 确认 建立连接（分配连接的资源）
  • FIN(finish)结束
  • ACK(acknowledment) 确认
  • 三次握手：
    • 1: 客户端向浏览器发起建立联机的请求 SYN码
    • 2：为客户端分配内存(建立联机) 回复ACK 确认码
    • 3：客户端向服务器发起确认码 ACK码

客户端 -SYN- 服务器 -ACK- 客户端 -ACK- 服务器

  • 四次挥手:
    • 1:客户端向服务端发起 结束码 FIN码
    • 2:服务器向客户端回写确认码 ACK码
    • 3:等待服务器确认所有数据都已经发送成功 FIN码
    • 4:客户端发送ACK 确认码 (关闭连接)

客户端 -FIN- 服务器 -ACK- 客户端 -FIN- 客户端 -ACK- 服务器

  • 转发和重定向
    • 转发:我找你借钱,你没有,我帮你找别人借,在借给你
      • 地址栏不会发生改变，原有的请求与响应对象依然存在
    • 重定向: 我找你借钱,你没有,我叫你找别人借
      • 地址栏会发生改变，原来的请求与响应对象，不存在了，只存在重定向后的请求与响应对象

小补充

  • http.STATUS_CODES 获取对应的常用常态描述
  • URL完整形式:http://127.0.0.1:8080/getOrder#abc?a=1
    • http：协议
    • 127.0.0.1 主机
    • 8080 端口
    • /getOrder 请求资源路径

    • 锚点

    • a=1 以键值对形式 查询字符串
  • 非完整形式就是更大的超集, 就是URI
