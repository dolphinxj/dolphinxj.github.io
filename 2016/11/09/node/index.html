javascript 和 nodejs 在运行环境的区别

1.JavaScript的运行环境是什么？

  • 浏览器（js解析引擎chrome：V8,完成解析,执行代码）
  • 通过nodejs平台在服务器运行

2.浏览器中 javascript 可以做什么？不可以做什么？

可以做：

  • 操作dom元素
  • 发送请求
  • 页面动态效果
  • ajax请求（XMLHttpRequest）
    不可以做：
  • 文件操作
  • 进程的操作
  • 接收请求
  • 数据库的操作

Chrome浏览器的实现结构

html/css : webkit布局引擎,完成布局和渲染
JavaScript : V8解析引擎,解析js代码并执行
中间层: 负责调配上下层以及硬件层: 网卡/硬盘
底层硬件

node的实现结构

基于以上结构，减去了DOM和BOM ，意思是运行环境没有document？window？
虽然少了BOM和DOM 但是不必担心页面的渲染。留下了V8 来按照ECMAScript来解析js代码并执行。

如何区分nodejs和浏览器环境？

什么是node？

nodejs是构建在V8引擎之上的一个运行时 平台
nodejs 基于V8 事件驱动 非阻塞IO模型 高效轻便

  • js语言来编码
  • 底层调用资源
  • 跨越平台

bower 前端资源
npm 前后端都有

nodejs可以做什么？

  • 服务器:
  • 命令行工具
  • 文件的操作
  • 进程的管理
  • 接受请求
  • 将数据持久化

nodejs和php有什么区别

  • 类型java/c/c++/php java –>服务器应用程序 web开发中要接收请求，还要服务器运行java代码 IIS apche
  • nodejs不需要服务器，本身这个平台就是服务器
  • 相同点：都具备后台语言能力 与 服务器

使用node的企业类型

  • 国外：Uber/linked in 把消息队列从 ruby 语言切换了node
  • 国内：BAT天猫
    • 处理高频率的访问

node应用场景

  • 多人游戏服务器
  • 基于浏览器的多人聊天室
  • 开发单页面程序
  • 基于json支持的api
    nodejs开发特点:
  • nodejs适合开发IO密集 （基于网络数据的输入和输出）
  • nodejs不适合CPU密集（频繁计算耗费CPU）

那天猫怎么使用nodejs？

CMD路径

  • 文件存在与当前的目录下 直接node 文件名
  • win + R 输入CMD（非nodejs不需要打开相关命令 全局下）
    • 绝对路径
    • 相对路径
    • 拼接路径
  • 推荐：shift + 鼠标右键 选择此处打开命令行窗口

命令行基本命令

  • dir 显示目录
  • cd 进入目录
    • cd / 当前盘符根目录
      • windows 中 /是代表当前盘符
      • Linux 中 /是总的根目录
    • cd .. 上级目录
  • 盘符切换 c:
  • 清屏
    • windows中 cls
    • powershell中 clear
  • ipconfig/all 获取本机IP信息
    • 有线：以太网适配器 IPv4地址…..192…
    • 无线：无线局域网 IPv4
  • shutdown -s -t 毫秒值 在多少毫秒内关机
  • shutdown -a 解除自动关机

工作中，服务器查看

  • 测试
    • ping + IP地址

组合键补充

  • WIN + E 我的电脑界面
  • WIN + L 切换用户
  • WIN + D 显示桌面
  • WIN + R 再输入services.msc 打开服务界面
    • regedit 开启注册表
    • mstsc(MS terminal service clinet) 开启远程连接

环境变量

查找机制是优先从当前目录下查找.exe（）或者.bat（批量处理文件）,找不到就找path

  • 方式1：我的电脑-右键属性-高级系统设置-高级-系统变量或用户变量中 修改path值
  • 方式2：定义一个系统环境变量 例：NODE_PATH 值：路径，再到系统变量中的path变量，添加一个%NODE_PATH%
    • 在win7中path变量各个目录地址以;分割
    • 首先找当前目录，然后找path变量，win7从右向左，win10从上到下查找

REPL(Read-eval-print-loop)环境操作

  • 通过node启动（node -v）
  • 在内部执行全局的操作
  • ★ 应用场景:测试API
  • 退出：ctrl+c 或 .exit

Global对象类别及作用域

内置对象 Math array window Object Date

global全局对象有：

  • global 相当于window对象 不常用
  • process 进程 描述当前程序运行的信息
  • module
    • 封装当前文件的对应信息 空文件是{}
  • console 打印
  • ★ __dirname 当前文件所在目录（文件夹名）
  • ★ __filename 当前文件的绝对路径
  • Buffer 是一个类

区分全局和伪全局对象

伪全局对象（每个输出的值不一样）
Buffer require exports module dirname filename

真全局
console process （一个进程 pid）

模块作用域

nodejs中的3M

NVM 管理nodejs 的版本管理工具
NPM 第三方资源库
NRM 辅助NPM 切换NPM的源

NVM

  • 首先卸载，如果不卸载会把当前的版本给NVM控制
  • 安装两个目录：
    • 设置nvm的目录 设置nodejs 的目录 通常配置在同级目录
  • nvm version 查看nvm的版本
  • nvm ls 显示当前版本列表 *表示当前
  • 安装nvm install 4.3[\.5] [32][64] [.5]不写会模糊匹配4.3以后版本
  • 下载 nvm uninstall 4.3.5
  • 切换版本 nvm use 4.3.5

nvm绿色版

  • 安装文件以后 修改文件名
  • 继续安装其他版本
  • 有了较新的版本 无法安装老版本
  • 安装完毕后，在电脑属性修改环境变量 path值
  • 重启命令行
